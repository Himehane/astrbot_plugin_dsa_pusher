# DSA推送器 (astrbot_plugin_dsa_pusher)

[English](README_EN.md)

AstrBot 插件，接收 DSA (DailyStockAnalysis) 通过 Webhook 推送的股票分析报告，自动转发到微信/QQ 等聊天平台。

最初基于 astrbot_plugin_daily_stock_analysis_adapter 重构，现已独立演化为功能更完善的 DSA 推送器。

## 工作流程

```
DSA 生成报告
     │
     ├── 自定义 Webhook → Markdown
     │                       │
     │                       ├── 文字模式 ── 语法清理（去##、**等）→ 整篇推送
     │                       ├── Markdown模式 ── 保留MD语法 → 整篇推送
     │                       └── 图片模式 ── 整张长图 / 按章节拆多张竖版小图
     │
     └── AstrBot Sender  → HTML（含样式）
                            │
                            ├── 文字模式 ── html2text 转 MD → 语法清理 → 整篇推送
                            ├── Markdown模式 ── html2text 转 MD → 整篇推送
                            └── 图片模式 ── 注入插件 CSS → 渲染图片
```

## 功能特性

- ✅ 接收 HTTP Webhook（兼容 Markdown / HTML 两种来源）
- ✅ **文字模式** — 自动清理 markdown 语法符号，方便复制转发
- ✅ **Markdown模式** — 保留完整 MD 语法，适合支持渲染的平台
- ✅ **图片模式** — 渲染为图片推送，阅读体验更好
- ✅ **表格卡片式展示** — 所有表格统一转为卡片格式，手机端阅读更直观
- ✅ **可选多图拆分** — 关闭=整张长图，开启=按章节拆多张竖版小图
- ✅ **多目标推送** — 同时推送到多个用户/群聊，自动检测平台
- ✅ **来源自适应** — HTML 自动转 Markdown，无需手动配置
- ✅ **全配置面板化** — Web UI 直接改配置，无需重启
- ✅ **聊天指令查询** — 微信内直接查行情、报告、复盘、管理自选股、控制推送开关

## 聊天指令

在聊天窗口直接发送以下指令（以 `/DSA` 前缀区分，避免与其它插件冲突）：

| 指令 | 功能 | 示例 |
|------|------|------|
| **/DSA 帮助** | 显示所有可用命令 | `/DSA 帮助` 或 `/DSA h` |
| **/DSA 任务 [n]** | 查看最近分析任务列表 | `/DSA 任务 10`（显示10条） |
| **/DSA 报告 [ID]** | 拉取指定任务报告全文 | `/DSA 报告`（最新）/ `/DSA 报告 abc123` |
| **/DSA 复盘** | 推送最新大盘复盘报告 | `/DSA 复盘` |
| **/DSA 自选行情** | 查看自选股实时行情 | `/DSA 自选行情` |
| **/DSA 历史分析 \<代码\>** | 查询个股历史分析报告 | `/DSA 历史分析 000001` |
| **/DSA 自选报告** | 批量推送所有自选股报告 | `/DSA 自选报告` |
| **/DSA 我的自选** | 查看当前自选股列表 | `/DSA 我的自选` |
| **/DSA 增加自选 \<代码\>** | 添加股票到自选列表 | `/DSA 增加自选 600519` |
| **/DSA 删除自选 \<代码\>** | 从自选列表移除股票 | `/DSA 删除自选 600519` |
| **/DSA 开启推送** | 开启自动推送通知 | `/DSA 开启推送` |
| **/DSA 关闭推送** | 关闭自动推送通知 | `/DSA 关闭推送` |
| **/DSA 推送状态** | 查询推送通知开关状态 | `/DSA 推送状态` |

## 安装方式

1. 将插件文件夹复制到 AstrBot 的 `data/plugins/` 目录下
2. 重启 AstrBot
3. 在 WebUI 中配置参数

## 配置说明

在 AstrBot 的 WebUI → 插件配置中进行设置：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `debug` | 调试模式：`true`=显示详细日志与目标ID，`false`=仅警告/错误 | `false` |
| `target_user_ids` | 推送目标列表（填裸 ID，代码自动补平台前缀） | `[]` |
| `output_mode` | 推送模式：`text` 文字 / `markdown` Markdown / `image` 图片 | `text` |
| `split_image` | 图片模式下是否按章节拆分多图（false=整张长图） | `false` |
| `webhook_path` | Webhook 路径 | `/stock-analysis` |
| `webhook_port` | 监听端口 | `8080` |
| `image_quality` | 图片质量 0-100（图片模式有效） | `85` |
| `device_scale_factor_level` | 像素缩放：`low` / `normal` / `high` | `normal` |
| `viewport_width` | 渲染视口宽度 px（图片模式有效） | `800` |

### 推送模式对比

| 模式 | 优点 | 适合场景 |
|------|------|---------|
| **文字模式 (text)** | 清理语法符号、可复制转发、体积小 | 手机端阅读、技术用户、需二次处理 |
| **Markdown模式** | 保留完整 MD 语法、支持渲染 | Telegram、支持 Markdown 的平台 |
| **图片模式 (image)** | 排版精美、阅读体验好 | 电脑端方便查看、非技术用户 |

### 多图拆分说明

图片模式下：
- **拆分关闭 (split_image=false)** — 整篇报告渲染为一张长图，保持完整
- **拆分开启 (split_image=true)** — 按 `##`/`###` 章节拆成多张竖版小图，手机滑动阅读更流畅

## 使用方式

1. 在 DSA 管理后台配置 Webhook URL：`http://你的AstrBot地址:8080/stock-analysis`
2. DSA 生成报告后自动推送到本插件
3. 插件根据配置处理内容并发送到目标用户/群聊

## 注意事项

1. 确保配置端口未被其他服务占用
2. 推送前需先向机器人发送任意消息建立对话上下文（部分平台需要）
3. 图片模式下若文件过大可调低 `image_quality`
4. 所有配置通过 WebUI 修改即可，无需编辑代码文件
